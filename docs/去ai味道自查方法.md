# 去 AI 味道自查方法

> 本文用于审校《架构之道》及未来技术写作时,自查并清除"AI 味道"。
>
> **AI 味不是"写错了",而是"写得太像 AI"**——语法零瑕疵、结构零缺陷、观点零风险,结果是信息增量趋近于零,文字失去了人类写作的"生命体征"。本文给出可操作的识别清单和去除方法。
>
> 整理自两份独立调研(2024–2026 年中英文来源),经交叉验证。文末附来源。

---

## 0. 先认识"AI 味"的本质

一句话定义:

> **AI 味 = 可预测性(低困惑度)+ 均匀化(低突发性)+ 安全感(观点零风险)的三重叠加。** AI 每次都挑概率最高的词、用最均衡的句长、走最安全的观点、套最工整的结构——结果是"好得没有个性"。

更深一层的重要洞察:**中文的 AI 味在很大程度上就是翻译腔**。每个字是中文、语法也合规,但句子骨架、主谓搭配、动词选择、段落节奏全是英文的,像是"先在英文句法里想清楚,再逐字换成中文"。所以去中文 AI 味,很多时候是在做"把英文句法骨架换回中文句法骨架"。

读者一眼看出的直觉来自:
- **过度完美**:用词规范、语法无瑕、标点一丝不苟;
- **整齐到失真**:一个比喻能拆出三个子比喻,工整到"不像人";
- **一本正经、情感扁平、修辞公式化**;
- **没有个人声音**:无观点跳跃、无真实情绪、无口语。

因此,去 AI 味的本质是反其道而行:**有意识地牺牲"安全与通用",换取"具体、有风险、像某个人写的"。** 把高概率的安全表达,换成有代价的具体表达。

## 1. AI 味的五大特征(识别)

### 1.1 结构:过度工整、机械对称

- **"总-分-总"三段式 + 机械小标题**:开头抛观点 → 列 3-5 个小标题 → 结尾"综上所述"收束。每个小标题内部又是同样的套路。
- **小标题过密、列表/加粗过密**:每个要点必加粗,几乎每段都用 bullet。
- **"滴水不漏"的完美感**:语法完美、标点一丝不苟,反而失去了人类写作的自然感。
- **TED 演讲体**:2025 年后的新套路,先抛金句、再升华,追求 drama 而非信息。

### 1.2 词汇:模板化套话与黑话

- **万能开头/结尾**:"在……的背景下""随着……的不断发展""众所周知""综上所述""由此可见""总而言之"。
- **互联网黑话(新八股)**:赋能、抓手、闭环、沉淀、打通、链路、颗粒度、对齐、迭代、复盘、底层逻辑、组合拳、降本增效。
- **学术腔概括词**:深入探讨、至关重要、不可或缺、举足轻重、系统性、多维度、全方位、范式。
- **英文 AI 词典**:delve、pivotal、intricate、underscores、showcasing、meticulous、notably、tapestry、landscape、testament、rich tapestry、navigate the landscape。
- **翻译腔**:"她是漂亮的"(直译 she is beautiful)而非"她很漂亮"。

### 1.3 句式:翻译腔对仗 + 长从句

- **对仗凑数**:"不是 A,而是 B""既……又……还……""不仅……更……""一方面……另一方面……"。判定:**同一篇里"不是…而是…"出现 2 次以上就触发 AI 味。**
- **"X 很 Y:"起手式**:"逻辑很清晰:""原因很简单:""结论很明确:""验证得很干净:"——冒号前先甩一个判断,冒号后再补内容。这种"先抛评价再补事实"的英文式骨架很 AI。
- **物理动作动词(英文直译,中文圈最新识别模式)**:把抽象思维说成物理动作——**接住/击穿/拆解/落地/收口/扛住/撑不住/不崩/打穿/收紧/推开/锋利**。如"这个论证被一击即穿""底层逻辑被打通了"。这些词单看正常,密度一高就是 AI 痕迹。
- **"进行 + 动词"(名词化)**:"进行讨论""进行分析""进行研究""进行优化"——动词被强行名词化再配"进行",是典型欧化。
- **词性后缀堆砌**(性/化/度/力):可操作性、数字化、智能化、颗粒度、渗透力、颗粒化——后缀连用来制造"专业感"。
- **框架句**:"通过……的方式""基于……的理念""在……的框架下"。
- **长从句**:一句话塞三四个分句,三个以上"的"连用,读起来绕口。
- **程度副词空泛量化**:"显著提升""极大地优化""全面支持""大幅度降低"。

### 1.4 逻辑:观点骑墙、信息密度低

- **观点零风险、两边都说**:永远平衡、中性、谁都不得罪。精辟概括:"语法零瑕疵、结构零缺陷、观点零风险、信息增量为零"。
- **对比修辞框架**:"not just X, but Y / 不仅……更是……"这种看似有锐度实则空洞的对仗来制造戏剧感。
- **把一句话撑成两千字**:靠排比、复述、过渡词注水,信息密度极低。
- **缺乏具体细节、举例雷同**:抽象概括多,找不到一个具体数字、一次踩坑、一段报错、一个真实场景。

### 1.5 指纹:统计特征(检测器原理)

| 指标 | 含义 | AI 文本 | 人类文本 |
|------|------|---------|---------|
| **困惑度(Perplexity)** | 下一个词的"意外度" | **低**(总挑概率最高的词) | 高(常有不预期用词) |
| **突发性(Burstiness)** | 句长/复杂度的变化幅度 | **低**(句长均匀、节奏单一) | 高(长短句交错、节奏跳跃) |

可观察的具体差异:
- **句长分布**:AI 集中在中等区间、方差小;人类长短交替,有 5 字短句和 50 字长句混排。
- **转折词密度**:AI 过度堆叠"因此/然而/此外/综上";人类靠语意衔接,连接词少得多。
- **具体 vs 抽象比例**:AI 偏抽象概括;人类偏具体细节(数字、案例、报错、场景)。

## 2. AI 词汇黑名单(自查:全文搜一遍)

> **原则**:不是绝对禁用,而是单篇出现频率过高 / 出现位置模式化就触发"AI 味"。能不用就不用,要用就换具体说法。

### A. 万能开头/过渡/结尾框架词(最该删)

| 词/短语 | 问题 | 替代 |
|---------|------|------|
| 在……的背景下 | 翻译腔 in the context of | 直接说时间/原因 |
| 随着……的不断发展 | 套路开场 | 删掉,直接进主题 |
| 众所周知 / 毋庸置疑 | 废话开场 | 删 |
| 首先 / 其次 / 再次 / 最后 | 机械编号 | 用"先…再…""一头是…另一头是…"或靠语意衔接 |
| 一方面……另一方面…… | 对仗凑数 | 拆成两句独立陈述 |
| 值得注意的是 / 值得一提的是 | AI 最爱的插入语 | 直接说事实,让读者自己判断 |
| 综上所述 / 总而言之 / 由此可见 / 总的来说 | 万能结尾升华 | 用一句具体结论或反问收尾 |
| 此外 / 另外 / 与此同时 | 高频连接词 | 用句号断开 |

### B. 互联网黑话/宏大叙事(最假)

赋能、助力、携手、共建、打造、构建、重塑、引领、驱动、护航、激活、释放、深耕、布局、生态、闭环、矩阵、抓手、势能、赛道、护城河、降本增效、数字化转型、新质生产力。

### C. 学术腔/概括词(技术文重灾区)

深入探讨 / 深入剖析 / 深入研究、至关重要、不可或缺、举足轻重、系统性、多维度、全方位、范式、范式转移、底层逻辑、顶层设计、核心要义、本质特征。

### D. 程度副词(空泛量化,技术文最致命)

显著(提升/下降)、明显、极大(地)、大幅(度)、充分、有效、高效、强大、完善、优越、卓越、优秀、良好、出色。

### E. 技术文专属黑话(把简单事说得很宏大)

优雅(的解决方案)、健壮(robust)、无缝(seamless)、开箱即用、一站式、轻量级、高性能、可扩展、现代化、下一代、智能化、自动化、标准化。

### F. 物理动作动词(中文圈最新识别的 AI 模式)

把抽象思维说成物理动作,密度一高就是 AI 痕迹:接住、击穿、拆解、落地、收口、扛住、撑不住、不崩、打穿、收紧、推开、锋利、咬合、咬死、兜住。

### G. 词性后缀堆砌(性/化/度/力,造专业感)

可操作性、数字化、智能化、颗粒度、渗透力、颗粒化、规范化、自动化、现代化、体系化、工程化、产品化。

### H. 英文 AI 高频词 → 中文映射

英文有这套 AI 词典(delve/tapestry/robust...),中文写作里见到对应的中文套话同样是 AI 味。英文写作里见到左列词也要警觉:

| 英文 AI 高频词 | 中文 AI 味对应 |
|---|---|
| delve into | 深入探讨 / 深入了解 |
| a rich tapestry of | 交织成……的画卷 / 丰富多彩的 |
| navigate the landscape | 在……格局中 / 应对……的复杂性 |
| in the realm of | 在……领域 / 范畴 |
| embark on a journey | 开启一段旅程 / 踏上……之旅 |
| this underscores | 这凸显了 / 彰显了 |
| leverage the power of | 借助……的力量 / 赋能 |
| robust / seamless / comprehensive / intricate | 强大的 / 无缝的 / 全面的 / 错综复杂的 |
| empower / enable / foster / catalyze | 赋能 / 使能 / 培育 / 催化 |
| synergy / paradigm / ecosystem / methodology | 协同 / 范式 / 生态 / 方法论 |
| it's important to note | 需要注意的是 |
| in today's digital landscape | 在当今数字化时代 |
| let's dive in | 让我们一探究竟 |
| in conclusion / in summary | 总之 / 综上所述 |

> 英文 AI 写作还有两个标志性指纹:**破折号(em dash —)滥用**、强行平行结构(not only X but also Y)。

## 3. 改写对照(AI 味 → 自然中文)

### 翻译腔对仗("不是…而是…")
- AI:❌ 这**不是**一个问题,**而是**一个机遇。
- 人:✅ 这其实是个机会。

### 框架开头
- AI:❌ **在数字化转型的背景下**,企业需要构建以数据驱动的产品迭代机制……
- 人:✅ 现在大家都在做数字化转型,产品迭代也得围着数据转……

### 万能结尾
- AI:❌ **综上所述**,我们需要在数字化转型中找到平衡。
- 人:✅ 说到底,转型这事得一步步来,急不得。

### 抽象大词堆砌
- AI:❌ AI **赋能**教育行业,**重塑**学习体验,**打造**全新**生态闭环**。
- 人:✅ AI 给教育带来了几个新工具,学习方式确实变了。

### 程度副词空泛量化(技术文重灾区)
- AI:❌ 该方案**显著提升了系统性能**,**极大地优化了用户体验**。
- 人:✅ QPS 从 2000 涨到 8500(4 倍多),首屏从 3.2s 降到 0.8s。

### 被动/物称主语(欧化)
- AI:❌ **数据的收集与分析被执行**,结果显示……
- 人:✅ 我们先把数据采回来分析了一遍,发现……

### 物理动作动词(把抽象说成物理)
- AI:❌ 这个论证被**一击即穿**,底层逻辑被**打通**了。
- 人:✅ 这个论证站不住脚,核心假设是错的。
- AI:❌ 方案 A **撑不住**高并发,让方案 B 来**兜底**。
- 人:✅ 方案 A 扛不了高并发,得靠方案 B 兜着。

### "进行 + 动词"(名词化)
- AI:❌ 我们对日志**进行了分析**,对结果**进行了讨论**。
- 人:✅ 我们分析了日志,讨论了结果。

### "X 很 Y:"起手式(先抛评价再补事实)
- AI:❌ **逻辑很清晰:** 因为 fork 要复制页表,所以实例越大越慢。
- 人:✅ 实例越大 fork 越慢,因为 fork 要复制整个页表。

## 4. 技术写作的特殊 AI 味问题

技术文章/技术书的 AI 味有 6 个特殊性,审校时重点查:

### 4.1 把简单机制说得过于宏大
一个 if-else 分支被包装成"智能决策引擎",一个缓存说成"高性能智能调度体系"。技术读者一眼穿帮。
**改法**:用机制本来的名字,少用"智能/自动/引擎/体系"这类放大词。

### 4.2 空泛量化,没有数字(技术文最致命)
"显著提升了性能""极大地优化了体验""支持高并发"——技术写作的价值就在于**可复现的数字**。
**改法**:每个性能声明必须配具体指标 + 测试条件。例:"QPS 从 2000 提升到 8500(4.25×),10 万并发下内存稳定在 2GB 以内,p99 延迟 < 50ms。"

### 4.3 代码注释 / README 的 AI 套话
- 注释废话:`// 这个函数用来做那个事情`、`// 参数是输入,返回值是输出`、`// 优雅地处理异常`(优雅是 AI 注释高频词)。
- README 套话:开头必是"🚀 一个现代化、轻量级、开箱即用、高性能的……",特性列表全是"✨ 优雅 / 🏗️ 健壮 / 🔒 安全 / ⚡ 高效"。
**改法**:注释只写"为什么",不写"是什么";README 砍掉形容词,放真实 benchmark 和使用示例。

### 4.4 模板化的"保姆级/手把手/一文读懂"结构
标题党 + 三段式 + 每节都有"小结",信息密度极低。技术书尤其忌讳——读者要的是知识,不是"学习路径安慰剂"。

### 4.5 过度使用专业术语堆砌
能用大白话说清的概念,偏要套一堆英文术语和缩写。
**改法**:首次出现给中英对照,之后用中文;复杂概念先用类比解释,再给术语。

### 4.6 缺乏真实的工程踩坑
AI 写的技术文几乎从不写"我试过 A 方案,在 X 场景下挂了,报错是……后来换成 B 才好"。
**改法**:加入真实工程经验——可量化结果、具体案例、踩过的坑。

## 5. 自查清单(写完逐项打勾)

### 结构层
- [ ] 是否每段都是"主题句 + 解释 + 举例 + 小结"的四段式?
- [ ] 是否满篇"首先 / 其次 / 再次 / 最后"?
- [ ] 小标题是否过于工整、字数接近、对仗排比?
- [ ] 结尾是否有"综上所述 / 总而言之 / 由此可见"式的强行升华?

### 句式层
- [ ] 句子长度是否高度均匀(没有突然的短句、感叹、反问)?
- [ ] 是否大量"在……的背景下""通过……的方式""基于……的理念"框架句?
- [ ] 是否有翻译腔对仗:"不是 A,而是 B""既……又……还……"?
- [ ] 是否一句话超过 40 字、三个以上"的"连用?

### 词汇层
- [ ] 是否高频出现黑名单里的"赋能 / 助力 / 抓手 / 闭环 / 深入探讨 / 至关重要"等词?
- [ ] 程度副词是否空泛:"显著提升 / 极大地优化 / 全面支持 / 大幅度降低"?

### 视角与细节层(技术文最重要)
- [ ] 全是抽象概括,找不到一个具体数字、一次踩坑、一段报错、一个真实场景?
- [ ] 是否有主观判断、价值取舍(而不是四平八稳谁都不得罪)?
- [ ] 比喻是否都是"像一座桥梁""如同一把钥匙"这类万能比喻?

### 情感层
- [ ] 语气是否过于均衡、四平八稳、谁都不得罪?
- [ ] 是否没有任何"我喜欢 / 我踩过坑 / 这玩意坑死我了"之类的主观颗粒度?

> **一条速判经验**:"结构严谨到可疑"本身就是信号——人类写作天然带"随机波动",AI 写作趋于平滑。

## 6. 一个自动化自查小脚本

对照上面的黑名单,可以跑一个简单的全文扫描,标出所有"嫌疑词"的出现次数和位置:

```bash
# 统计黑名单词汇出现次数(按词频降序)
python3 << 'EOF'
import re, glob
from collections import Counter

# 黑名单(按类别)
blacklist = [
    # A 框架词
    "在.{0,8}的背景下", "随着.{0,8}的不断", "众所周知", "毋庸置疑",
    "首先", "其次", "再次", "最后", "一方面", "另一方面",
    "值得注意的是", "值得一提的是", "综上所述", "总而言之",
    "由此可见", "总的来说", "与此同时",
    # B 互联网黑话
    "赋能", "助力", "抓手", "闭环", "沉淀", "打通", "链路",
    "颗粒度", "对齐", "迭代", "复盘", "底层逻辑", "组合拳",
    "降本增效", "数字化转型",
    # C 学术腔
    "深入探讨", "深入剖析", "至关重要", "不可或缺", "举足轻重",
    "多维度", "全方位", "范式", "顶层设计", "核心要义",
    # D 程度副词
    "显著", "极大", "大幅度", "充分", "高效",
    # E 技术黑话
    "优雅", "健壮", "无缝", "开箱即用", "一站式", "高性能",
    "现代化", "智能化",
]

counter = Counter()
locations = {}
for f in sorted(glob.glob('chapters/*/chapter.md') + glob.glob('chapters/0*.md')):
    for i, line in enumerate(open(f), 1):
        for w in blacklist:
            for m in re.finditer(w, line):
                counter[w] += 1
                locations.setdefault(w, []).append(f"{f.split('/')[1]}:{i}")

print(f"{'词':<12} {'次数':<5} 出现位置(前3处)")
print("-" * 60)
for w, n in counter.most_common():
    locs = locations[w][:3]
    print(f"{w:<12} {n:<5} {', '.join(locs)}")
print(f"\n共 {sum(counter.values())} 处嫌疑词, {len(counter)} 种")
EOF
```

跑完看输出:**高频出现的(尤其是单篇超过 3-5 次)优先改**;出现在"开头/结尾/小标题"这种模式化位置的优先改。

---

## 6.5 一句话判据:看密度,不看单点

这是全文最重要的判断方法,单列为一条:

> **单个成语、单个对仗、单个套话不算 AI 味;反复堆叠、换到任何话题都成立、跟前后内容没关系的,才是。**

换句话说:不是某个词"有罪",而是"这个模式出现了多少次、出现在多么模式化的位置"。判定 AI 味看的是**密度和模式化程度**:
- 一篇文章出现一次"综上所述"——正常。
- 一篇文章"首先…其次…再次…最后…综上所述…"——AI 味。
- 一处用"赋能"——可能正好用得对。
- 一段里"赋能 / 助力 / 打造 / 闭环"——AI 味。

所以自查的核心动作不是"这个词能不能用",而是**统计这个词/句式在本篇的密度和位置分布**,密度异常的才改。

---

## 7. 一句话总结

> **去 AI 味的核心原则:把"高概率的安全表达"换成"有代价的具体表达"——删框架词、拆长句、把抽象大词和空泛量化替换成具体数字与真实踩坑,让文字重新带上有风险的"个人选择"。**

---

## 8. 去 AI 味 Prompt 模板与工作流

### 8.1 用 AI 去 AI 味(慎用,见 8.3)

最有效的方法不是"去 AI 味",而是**注入你自己的味道盖过 AI 平均味**——用"我/你"代替"笔者/用户/人们",加入真实故事、具体案例、失败经历,允许表达立场、反讽、自嘲。下面是两个可用的 Prompt:

**通用去味模板:**
```
你是一位精通中文写作风格的编辑。请把下面的 AI 生成文本改得更自然、更像人写的:
1. 删除"首先/其次/最后""综上所述""值得注意的是""众所周知"等 AI 标志词;
2. 长短句交错,每 3 个长句后接一个短句,打破均匀节奏;
3. 用具体数字、案例、报错替换"显著提升""极大地优化"等空泛量化;
4. 把"进行讨论/进行分析"还原为"讨论/分析";把物理动作动词(击穿/落地/兜住)换准确说法;
5. 允许第一人称、判断、偶尔口语;保留原意和关键信息不变。
```

**场景化模板(更狠):**
```
请以第一人称、务实而克制的口吻重写这段技术文字。要求:
- 禁止出现"赋能/抓手/闭环/深入探讨/优雅/健壮/无缝"等词;
- 每个"显著/极大"必须配一个具体数字,否则删掉该评价;
- 把抽象主语换成具体动作或对象;把被动句尽量改主动句;
- 砍掉每段的过渡废话开头("接下来我们讨论…"),让段首句即论点。
```

### 8.2 个人"写作声音档案"(比一次性 Prompt 更有效)

维护一份自己的写作风格 Skill,记录口味、忌口、禁词。内容包括:

- **Tone(语气)**:务实、克制、有判断?还是温暖、轻松?
- **Rhythm(节奏)**:偏好长句还是短句?爱用反问吗?
- **Vocabulary(词汇)**:偏好什么词、讨厌什么词(禁词表)?
- **Perspective(视角)**:第一人称经验为主,还是客观观察?
- **Values(价值观)**:透明、质疑、极简?

每次让 AI 写作前喂入这份档案,AI 产出的初稿就自带你的味道,后续去味工作量大减。

### 8.3 重要提醒:别只靠"去味 Prompt"

去味 Prompt 能改掉表层套话,但改不掉**信息密度低、缺乏真实细节**这个根本问题。最有效的去 AI 味永远是人手的两个动作:

1. **加具体**:数字、版本、案例、命令、报错、配置、个人经历——这些是 AI 写不出来的硬证据。
2. **敢有立场**:删掉骑墙的"一方面…另一方面…",该下结论就下结论。

> 经验:**AI 出初稿 → 人工二次创作**,而不是"AI 出稿 → AI 去味"。后者只是在概率空间里打转。
