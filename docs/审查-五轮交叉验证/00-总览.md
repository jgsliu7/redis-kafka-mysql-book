# 五轮交叉验证 · 总览

> 《架构之道》去 AI 味 + 出版合规审查。**5 轮 × 每轮多 Agent 独立审查 + 交叉印证**，每个 CONFIRMED 问题均由 **≥2 个 Agent 一致认定**。本文件是五轮结果的总合并与优先级排序，各轮明细见 `round-1` ~ `round-5`。

## 一、怎么做的（方法论）

**调度原则**：我只调度，不亲自检测。每轮派多个独立 Agent 各自审查同一范围，取**交集**为 CONFIRMED；单 Agent 命中的"候选"再派**独立验证员**复核（读原文/查字节/WebSearch 官方文档），凑齐 ≥2 才升级，误报则剔除。

| 轮次 | 主题 | Finder Agent | 验证员 | CONFIRMED | 驳回 |
|------|------|-------------|--------|-----------|------|
| 1 | 去 AI 味 | ai-writing-detector + tech-editor | general-purpose | **21** | 10 |
| 2 | 文字校对/术语 | typo-grammar + term-consistency + general-purpose | general-purpose（字节核查） | **20** | 4 |
| 3 | 技术事实 | fact-consistency + number-provenance + command-runnable | general-purpose（WebSearch） | **15** | 5 |
| 4 | 交叉引用/图表 | crossref-validity + svg-semantics + general-purpose + architect-reviewer | general-purpose（读 SVG） | **8** | 6 |
| 5 | 出版合规/结构 | compliance + structure-pedagogy + general-purpose + interviewee-reader | general-purpose | **20** | 1 |
| | | **14 种 Agent，共 21 次调度** | | **合计 84 条**（去重后见下） | 26 |

**门槛严格执行**：没有一条 CONFIRMED 是只靠一个 Agent 的。冲突项（如引号字符、RDB version、商标齐全度）全部由验证员查字节/官方文档裁定。

## 二、交叉印证的价值：挡下的单 Agent 误报

"不要相信任何一个 Agent"这条规矩在五轮里**每次都拦下了会引入的错误修复**：

| 轮 | 单 Agent 误报 | 若信单 Agent 会怎样 | 谁挡下的 |
|----|--------------|--------------------|---------|
| 2 | general-purpose 报"半角分号残留 7 处" | 批量把正确的全角分号改成半角 | 验证员字节核查：7 处全是全角 0xff1b |
| 2 | general-purpose 报"短暂双写"应改"脑裂写" | 把正确的"双写"改成错误的"脑裂写" | 验证员：双写对槽迁移准确，脑裂写才错 |
| 3 | （候选）"单核十万 QPS"过时应改 | 把准确的"命令执行单核"改没 | 验证员：命令执行仍单核，措辞准确（KEEP） |
| 4 | architect 报 fig-8-2 "RDB version 0010"是二进制错 | 改一个本来正确的标注 | 验证员查 RDB 规范：是 ASCII "0010"=v10，正确 |
| 5 | general-purpose 报商标声明"齐全规范" | 漏修 ZooKeeper/InnoDB 缺失 | compliance + 验证员实证：ZooKeeper 25 次/InnoDB 125 次未声明 |

> **结论**：单 Agent 既有假阳性（半角分号、脑裂写、RDB version）也有假阴性（商标齐全）。≥2 交叉印证是唯一可靠的筛子。
