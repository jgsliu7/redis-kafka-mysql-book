# 《架构之道：从 Redis、MySQL、Kafka 看软件设计的共性》

## 作者初衷

> 多年前，吾观 Redis、MySQL、Kafka 三大软件，世人皆学其用法、配置、优化，却鲜有人从架构视角探寻其共性。
> 
> 三者皆为优秀软件，然解决之问题多有相通：
> - 如何优雅启动？
> - 如何优雅关闭？（数据存储类软件不可如断电般粗暴）
> - 内存与磁盘之关系如何处理？（求快必用内存，然内存与磁盘如何协同？）
> - 分层架构如何设计？（存储层、逻辑层、交互层之关系）
> - 安全如何保障？
> - 集群如何构建？
> - 磁盘文件格式如何设计？（三者各有妙法）
> - 集群间数据如何同步？
> 
> 万法归一，皆为特性取舍。此书旨在透过现象看本质，从架构高度理解软件设计之道。

## 版本约定

本书分析基于以下软件版本：

| 软件 | 版本 | 说明 |
|------|------|------|
| **Redis** | 7.x | 稳定版，包含多线程 IO、新集群特性 |
| **MySQL** | 8.0.x | 当前主流稳定版，包含新版 redo log、原子 DDL 等 |
| **Kafka** | 3.x | KRaft 模式成熟版本，逐步去除 ZooKeeper 依赖 |

> 注：版本特性会有演进，但核心架构思想具有长期稳定性。
