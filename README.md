# 《架构之道：从 Redis、MySQL、Kafka 看软件设计的共性》

## 关于本书

多年来，我们一直在用 Redis、MySQL、Kafka 这三款软件。大多数资料教的是用法、配置、调优，却很少有人从架构的视角去寻找它们的共性。

这三款都是优秀的软件，但它们解决的问题多有相通之处：

- 如何优雅地启动与关闭？（数据存储类软件不能像断电那样粗暴地停）
- 内存与磁盘的关系如何处理？（要快就得用内存，可内存与磁盘怎么协同？）
- 分层架构如何设计？（存储层、逻辑层、交互层之间的职责划分）
- 安全如何保障？
- 集群如何构建？
- 磁盘文件格式如何设计？（三家各有妙法）
- 集群之间数据如何同步？

万法归一，本质上都是特性的取舍。这本书想做的，是透过现象看本质，从架构的高度去理解软件设计之道：不把这三款当成孤立的工具，而当成对同一组问题的三次不同回答。

## 版本约定

本书分析基于以下软件版本：

| 软件 | 版本 | 说明 |
|------|------|------|
| **Redis** | 7.x | 稳定版，包含多线程 IO、ACL、Function、新集群特性 |
| **MySQL** | 8.0.x | 当前主流稳定版，包含新版 redo log、原子 DDL、组复制 |
| **Kafka** | 3.x | KRaft 模式成熟版本，逐步去除 ZooKeeper 依赖 |

> 版本特性会持续演进，但核心架构思想具有长期稳定性。
