# 终审流程 Playbook（可复用）

> 任何中文技术书（或长技术文档）投稿/发布前的**最后一轮审查**。本文件是入口和流程编排，把方法论（01）、审查 agent 人设（02）、工具脚本（03）、实战样本（04）串成一条可执行的流水线。
>
> 整理自《架构之道：从 Redis、MySQL、Kafka 看软件设计的共性》终审实战（2026-06，6 轮 commit）。适用于任何"已完成初稿、要做投稿前终审"的技术内容。

---

## 0. 两条铁律（开工前先内化）

### 铁律 1：语义类 ≠ 词汇类，检查方法不同
- **词汇类**（黑名单词：赋能/抓手/至关重要/显著/极致…）→ 可以 `grep` + 看密度。
- **句式/逻辑类**（对仗翻译腔/机械排比/判断悬空/收口标签/空泛拔高/三栏镜像…）→ 是**语义**问题，**必须派 agent 逐段通读**，**严禁用 grep 关键词自证"已清零"**。
  - 原因：同一类问题能换无数种措辞（"不是A而是B" → "看似…实际…" / "不再…而是…" / "X而非Y" / "不仅…更…"），grep 只匹配已知词，**必然漏报**；更糟的是 grep 命中 0 会给"虚假的零残漏信心"。
  - 这是本流程最重要的教训，详见 `01-方法论/去ai味道自查方法.md` §1.3q「检查方法」。

### 铁律 2：改字词不改意思
- 去 AI 味只改表达方式，**不增删信息、不动事实/数字/命令**。
- **绝对保留**：作者第一人称 + 真实案例（踩坑/事故/选型争论）、取舍段的判断锋芒、结构性锚点比喻（如"真相之源/骨架/状态机"）。这些恰恰是最不像 AI 的部分，删了反而坏事。

## 1. 流水线（5 阶段）

```
阶段0 确定性基线  →  阶段1 多维度只读审查  →  阶段2 分区修复  →  阶段3 回归+对抗复检  →  阶段4 飞轮沉淀
（快，先跑）        （并行 agent，语义读）     （并行 agent，按文件独占）  （抓回退+验证）       （补清单）
```

### 阶段 0：确定性基线（快，先跑）
拿到硬清单，聚焦后续 agent：
- **回归断言**（`pub_assertions.py` 式）：把已核实为正确/已清理的事实与术语编码成 `grep`/`count` 断言，防"修了又回"。
  - ⚠️ 断言只锁**字串/事实类**回归（如 `三家=0`、`broker→Broker`、`一千万倍`）；**语义类属性**（如"CAP 是否 hedged""AI 味是否消除"）**不能用 grep 断言锁死**，权威检查靠阶段 1 的 agent 通读。
- **几何/格式审计**（如 `svg_audit.py`）：插图越界/穿字/连接。
- **词汇类黑名单扫描**（如 `去ai味道自查方法.md` §6 脚本）：只看词汇类密度。
- **能否构建**（`build_html.py`）。

### 阶段 1：多维度只读审查（并行 agent，**无 Write 工具 = 零冲突**）
派一组专精 agent，每个负责一个维度，**全书逐段通读**（语义读，不是 grep）。维度清单（按本书实战，可按项目裁剪）：

| 维度 | 查什么 | 对应 agent 人设（02） |
|------|--------|----------------------|
| **AI 味** | §1.3 全家句式/逻辑病理 + 逆向发现新型 | ai-writing-detector |
| 文字三校 | 错别字/语病/"的地得"/半角残痕 | typo-grammar |
| 术语一致性 | 同概念一个叫法、缩写首现全称 | term-consistency |
| 交叉引用 | "详见第N章/图N-M"指向正确、三一致 | crossref-validity |
| 事实一致 | 同机制/判断/参数跨章不矛盾 | fact-consistency |
| 命令可运行 | 命令/SQL/参数在基线版本存在且对 | command-runnable |
| 数字溯源 | 裸数字有出处/前提/不过时 | number-provenance |
| 结构教学 | 前置不倒置、小结有新判断、导读兑现 | structure-pedagogy |
| 阅读体验 | 钩子/节奏/留存 | tech-editor |
| 插图语义 | 箭头方向/流程顺序/数值与正文一致 | svg-semantics |
| 合规 | 商标/参考文献/版权要素 | compliance |

产出：各维度 findings（`file:line | 类 | 原文 | 为什么 | 建议改法`）。

### 阶段 2：分区修复（并行 agent，**按文件独占 = 零冲突**）
- 把 findings 按**文件**路由。每个文件（或文件组）划给**唯一** fix agent（独占），它同时处理该文件的所有维度 findings。
- 每个 fix agent 带：① 该文件 findings；② 风格约束（铁律 2）；③ 明确的 **FIX / SKIP 清单**（borderline 的标 SKIP，避免过度编辑伤作者声音/真实信息）；④ 全局术语映射表（术语/crossref 类给所有 agent 同一份，保证一致）。
- agent 用 `Edit` 精准替换，**不要 `Write` 覆盖整文件、不要整节推倒重写**；偏好"删"而非"重写"；打破对称**只动一家**（三家排比不要重写成新的对称）。

### 阶段 3：回归 + 对抗性复检（fix→verify 闭环）
- **重跑阶段 0 断言**：抓 fix agent 引入的回退（典型：重新冒出"三家"、破折号超标、绝对 CAP 措辞）。
- **对抗性复检**：派 agent 复检改动最重的几处——重写读着自然吗？删句后段落衔接断了吗？有没有**引入新 AI 味**？整章是否达标？
- 视觉复检：插图几何报警用 `svg_render.py` 渲 PNG → `analyze_image` MCP 真看图裁定（z-order/采样盲区会误报）。

### 阶段 4：飞轮沉淀（每轮清单更丰富）
- **新病理** → 命名（像语言学家）→ 补进 `去ai味道自查方法.md` §1.3x（定义 + 3+ 实例 + 通用改法）。
- **新检查方法教训** → 补进方法论（如"语义类不能 grep""回归断言只锁字串类"）。
- **新核实的事实/术语** → 补进 `pub_assertions.py`。
- 发现 → 沉淀 → 用更丰富的清单再发现更深层。每轮飞轮转一圈，清单加厚一层。

## 2. 标准速查表（维度 → 标准 → 工具/agent → 方法）

| 维度 | 标准 | 工具 / agent | 方法 |
|------|------|-------------|------|
| AI 味 · 词汇类 | §A–§H 黑名单 | §6 脚本 | grep + 看密度 |
| AI 味 · 句式/逻辑类 | §1.3c–§1.3q | ai-writing-detector | **语义通读** |
| 文字三校 | 零错别字/语病/半角 | typo-grammar | 通读 |
| 术语 | 全书一个叫法 | term-consistency | 通读 + 映射表 |
| 交叉引用 | 指向正确/三一致 | crossref-validity | 通读 + 建索引 |
| 事实一致 | 跨章不矛盾 | fact-consistency | 通读 |
| 命令/参数 | 基线版本可运行 | command-runnable | 通读 + 官方文档 |
| 数字 | 有出处/前提 | number-provenance | 通读 + 溯源 |
| 插图 · 几何 | 不越界/穿字 | svg_audit.py | 脚本 |
| 插图 · 语义 | 箭头/流程/数值对 | svg-semantics | 通读 + 源码 |
| 合规 | 商标/参考文献/版权 | compliance | 通读 |

> 一句话：**词汇类 + 几何类 + 字串事实类 → 工具/脚本；句式/逻辑/语义类 → agent 通读。** 别用错工具。

## 3. 实战常见坑（踩过的，记此防再踩）

1. **关键词验证语义类**：用 `grep "远超/上限"` 报"0 残漏"是虚假安全感——换措辞的实例（"最好的/触顶/体积更小/各自贴合"）grep 全漏。语义类必须 agent 逐段通读。（§1.3q 教训）
2. **三栏镜像是最顽固 AI 味**：三家软件在小结首句/对比段的完全对称分号串，**没有任何触发词**，多轮验证全靠它"没黑名单词"漏网。只能语义通读抓。改法：保留共因/规律一句，三家只动一家打破镜像。（§1.3k 实证）
3. **CAP/归类等语义属性不能用 grep 断言锁死**：agent 换个说法（"属于 AP"/"定位 AP"）就绕过断言。权威检查 = agent 通读。（`pub_assertions.py` CAP 注释）
4. **修复 agent 会引入回退**：去 AI 味重写时可能重新冒出"三家"、破折号超标、绝对 CAP 措辞。阶段 3 断言专门抓这个。
5. **对抗性复检会抓到你自己的改坏**：fix agent 删句后衔接断裂、或重写引入新对仗——必须 fix→verify 闭环，别信"改完就行"。
6. **过度编辑伤内容**：borderline（作者第一人称案例/取舍锋芒/结构性锚点/有真实信息的排比）要 SKIP。去 AI 味不是把书改平淡。
7. **SVG 几何脚本的盲区**：`svg_audit.py` 的 `[C]线穿字`/`[D]穿框` 有 z-order 遮挡和采样阈值盲区，会误报。命中必须视觉裁定：`svg_render.py` 渲 PNG → `analyze_image` MCP 真看图。
8. **断言只锁字串类**：`pub_assertions.py` 是"已知坏字串没回弹"的快速兜底，**不是**"该属性已验证"。语义属性验证靠 agent。

## 5. 一页纸速跑（复制到他项目时）

1. 把 `02-审查Agent人设/` 的 agent 人设装上（或按本项目维度裁剪）。
2. 写本项目的 `pub_assertions.py`（把已核实事实/术语编码成 grep 断言）+ 跑几何/构建基线。
3. 派阶段 1 的只读审查 agent 矩阵（按 §1 维度表），**通读**，收 findings。
4. 按文件分区派阶段 2 fix agent（独占、带 FIX/SKIP 清单 + 术语映射）。
5. 重跑断言 + 派阶段 3 对抗复检 agent。
6. 新病理/教训补进方法论 + 断言（阶段 4 飞轮）。
7. 交付构建产物（HTML/PDF）。

*本 playbook 随实战持续修订。每跑一轮终审，把新教训补进 §3 常见坑、新病理补进 `去ai味道自查方法.md`。*
