# 飞轮文风审查方法论

> 本文档描述一套**可复用的文风质量迭代方法**——通过"逆向发现 → 沉淀 → 检查 → 再发现"的飞轮循环,系统性地发现和修复写作中的不自然句式。适用于任何中文(或可迁移到其他语言)长文项目:技术书、系列博客、产品文档等。
>
> 本方法在《架构之道》一书中经过 4 轮实战验证,从每轮修复 70 处收敛到 0 处,证明方法有效且可收敛。

---

## 1. 核心问题:为什么需要飞轮?

传统的文风审查是**正向检查**:拿一份黑名单(AI 味词表/翻译腔清单),扫全文,命中就改。问题是:

- **只能抓已知的**。黑名单里没有的模式,扫多少遍都扫不出来。
- **黑名单靠人工积累,永远不全**。每本书/每个作者有自己的"盲区怪癖",通用清单覆盖不到。
- **发现一个修一个,没有系统**。作者或编辑读到某句觉得怪→分析→改→下次又遇到另一种怪→再分析。没有积累、没有加速。

飞轮解决的核心问题是:**怎么系统性地发现你自己都不知道的"盲区怪癖"?**

---

## 2. 飞轮的四步循环

```
    ① 发现(逆向)
   ↓
    ② 沉淀(补进清单)
   ↓
    ③ 检查(正向,用新清单扫全书)
   ↓
    ④ 再发现(逆向,找更深层的)
   ↓
    ... 每轮清单更丰富,发现更深层
```

### ① 发现:逆向发现(核心步骤)

**不用黑名单扫,而是让 agent 读完一章,找出"不含任何已知黑名单词、但读着就是不对"的句子。**

判定标准——**四测试**:

| 测试 | 判定 |
|------|------|
| **朗读测试** | 读出声来,像自然中文,还是像翻译/AI 生成? |
| **白板测试** | 一个资深工程师在白板前给同事讲,会**原样说**这句话吗? |
| **一遍读懂测试** | 第一遍读能抓住意思,还是要回头重读? |
| **不在清单测试** | 不含已知黑名单词,但就是别扭——底层是什么句式问题? |

**关键操作**:不报"第 X 行这句话怪",而是找 3+ 个同类,**命名成新的句式病理类别**(像语言学家给语言现象命名),给定义 + 实例 + 改法。

**prompt 模板**:

```
你是句式病理学家。任务:在一章正文里,发现现有检查清单覆盖不到的、新型的不自然句式。

【先读已知清单】(你的项目的 AI 味文档/风格指南)
这些里面已经列了的模式,不要再报。你的任务是找它们没覆盖到的。

【"不自然"的判定——四测试】
对每个句子问:
1. 朗读测试:读出声像自然中文,还是像翻译/AI?
2. 白板测试:工程师在白板前给同事讲,会原样说这句话吗?
3. 一遍读懂测试:第一遍能抓住意思吗?
4. 不在清单测试:不含黑名单词,但就是别扭——底层是什么句式问题?

【怎么报——归类命名,不是逐句罗列】
找 3+ 个同类怪句 → 命名 → 定义(共性特征)→ 3+ 实例 → 改法原则。
```

### ② 沉淀:补进清单

每个新发现的模式类别,经过验证(3+ 实例?真新?),**写进项目的 AI 味自查文档**。格式:

```markdown
### §X.XX [模式名]

**定义**: 这类问题的共性特征是什么。
**判定信号**: 怎么识别(标志性句式/词/结构)。
**改法**: 怎么改(原则 + before→after 示例)。
**注意**: 哪些情况不算(避免误报)。
```

### ③ 检查:正向修复

用更新后的清单(原黑名单 + 新发现的模式),派 agent 逐章修复。**同时做回归检查**(见下)。

### ④ 再发现:下一轮

清单变长后,agent 的"已知"更丰富,被迫去找**更深层的句式问题**。如果找不到——**报告"收敛"**,这是合法且有价值的结论。

---

## 3. 什么时候该停(收敛判据)

飞轮不是无限跑的。当以下信号出现时,说明已收敛:

| 信号 | 含义 |
|------|------|
| 修复量从 70 → 50 → 35 → 14(指数递减) | 问题快修完了 |
| 新模式发现:6 → 1 → 0 → 0(第三轮归零) | 没有新的了 |
| 10/10 章显式报告"收敛" | agent 自己确认找不到新问题 |

**实战经验**:2 轮覆盖 ~90% 的问题,第 3 轮确认收敛,第 4 轮冗余(可省)。

---

## 4. 回归检查(每轮必做)

agent 在修写时会引入**回归**(把已经修好的东西改回去)。这是飞轮的固有成本,必须在每轮后做**确定性回归复查**:

```bash
# 回归检查项(用确定性脚本,不用 agent)
python3 -c "
import glob
t = ''.join(open(f).read() for f in glob.glob('chapters/*.md'))
print('三家:', t.count('三家'))        # 应=0(已清理的词)
print('破折号:', t.count('——'))       # 应≤个位数(已收敛的标点)
print('真相之源:', t.count('真相之源')) # 应>0(不应被删的锚点)
print('骨架:', t.count('骨架'))        # 应>0
print('取舍:', t.count('取舍'))        # 应>0
"
```

**原则**:**事实/字串类**回归用一个确定性 grep/count 脚本检查(如 三家=0、破折号≤个位数、锚点>0)。不依赖 agent 判断——agent 是发现新问题的,不是检查旧问题的。**但语义类回归例外**(如 CAP 是否 hedged、AI 味句式是否回弹):agent 换个说法就能绕过 grep,这类必须派 agent 逐段通读复查(见 docs/去ai味道自查方法.md §1.3q)。

## 5. 在你的项目里怎么用

### 前置条件

1. **有一份 AI 味/文风自查文档**(哪怕只有几条)。这是飞轮的起点——agent 需要知道"哪些已经知道了"。
2. **有一份风格指南**(定义"正常"是什么)。即使只有几句话也行。
3. **有 agent 编排能力**(并行 fan-out + pipeline)。

### 启动第一轮

```javascript
// Workflow 脚本结构(伪代码)
const CHAPS = [/* 你的章节列表 */]

const results = await parallel(CHAPS.map(c => () =>
  agent(`${逆向发现prompt}\n\n【本次】${c.name}: ${c.path}`)
))

// 汇总 → 去重 → 验证 → 补进清单文档
```

### 每轮之间

1. **确定性回归检查**(grep/count,不用 agent)
2. **修回归**(确定性脚本或手动)
3. **重建产物**(HTML/PDF 等)
4. **提交**
5. **启动下一轮**(agent 拿到更新后的清单)

### 停止条件

修复量连续两轮递减到个位数 + 新模式发现归零 → 停。

## 6. 本项目的实战数据

《架构之道》(10 章技术书,约 20 万字)的飞轮数据(四轮类别级 + 第五轮逐句穷尽):

| 轮次 | 修复量 | 新模式 | 收敛章 | 信号 |
|------|--------|--------|--------|------|
| 第一轮 | 70 处 | 6 类 | 0/10 | 发现高峰 |
| 第二轮 | 53 处 | ~1 微类 | 7/10 | 收敛中 |
| 第三轮 | 35 处 | 0 | 8/10 | 强收敛 |
| 第四轮 | 14 处 | 0 | 10/10 | 全收敛(类别级) |
| **第五轮(逐句穷尽)** | **~190 处** | **8 类(+3 零星)** | **1/10 显式收敛** | **逐句比类别轮更深** |
| **总计** | **~362 处** | **15 类** | — | — |

> **第五轮的关键发现**:第四轮已"全收敛"的判断是**类别级**的(逐类扫全书)。换成**逐句穷尽**(每章派 4 视角 agent 把每句话过四测试)后,又挖出 ~190 处和 8 类新模式——证明"收敛"是相对当前检查粒度而言的,**降低粒度就能发现更深层的问题**。这一轮 Ch1 引言和 Ch5 分层架构是重灾区(分别 22、50 处),说明越是结构性、论说性强的章节,翻译腔和收口模板残留越多。详见 `去ai味道自查方法.md` §1.3i–p。

发现的 15 类新句式病理(均为通用黑名单未覆盖):

1. **机械三连排比收束句**(段末三句结构完全对称)
2. **判断悬空句**("这是X的根因"缺因果扣合)
3. **被动塑造腔**("被X塑造" = shaped by 直译)
4. **连环设问堆叠**(3+ "为什么"悬空不答)
5. **商品化隐喻泛滥**("买的是/防的是"密集排比)
6. **冒号链式收口**(一句话 2-3 个冒号堆结论)
7. **过度抽象压缩+混合隐喻**("范式对应模式,根在判断题上")

第五轮(逐句穷尽)新增 8 类:

8. **段末抽象收口标签**(给已讲清的事实贴抽象名词:根因/动因/依据/价值/底座/形态)
9. **抽象名词主语 + 带来/换来**(翻译腔施事错位,"这个假设带来的好处是…")
10. **对照框架词与三家横比套句密度**("完全不同"开场 + 横比三连同构分号串)
11. **物理动词簇密度**(压/拉/塞/推/逼/搬/撑,单章 4+ 处累积翻译腔)
12. **主动式被动塑造腔**("三条要求把格式锁在骨架上"——§3 的主动变体)
13. **自复述收束**(段尾换词重砸段首论点,信息增量接近零)
14. **结构预告与公式化引导句密集**("本节按 N 个板块展开"+"理解 X 关键是抓住 Y")
15. **自定义结构锚的同节模板复用**(区分"心智锚点"保留 vs "模板复用"打散)

> 另有 3 类零星发现(单章孤例):倒装补名断句、动宾硬搭、夸饰性类比收口。详见 `去ai味道自查方法.md` §1.3i–p。

### 给其他项目的建议

1. **先建一份最小 AI 味文档**(哪怕只列 10 个词),作为飞轮起点。
2. **跑 2 轮飞轮**就能覆盖 ~90% 的问题。
3. **每轮后做确定性回归检查**——这一步不能省,agent 会引入回归。
4. **把新发现写进文档**——飞轮的价值在于积累,不在于单次修复。
5. **第三轮以后是冗余**,除非项目特别大(>50 万字)或风格特别复杂。
