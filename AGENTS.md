## Agent 工作方法论(本项目验证有效)

### 1. 多视角并行审查(每图/每章 3 个 agent)

**模式**:对一个审查对象(一章、一张图),派 3 个 agent 从不同视角独立审,交叉汇总。

**实例**:
- SVG 插图审查:布局 / 文字 / 线条 三视角。
- 章节文风审查:AI 味 / 翻译腔 / style-guide 合规 三视角。
- 选题审查:目标读者(2-3年) / 资深架构师(15年) / 技术编辑 三人设。

**为什么有效**:单视角容易漏;3 个独立视角交叉后,漏报率显著下降。agent 之间不需要沟通,各自独立判断,由负责人(你)综合。

### 2. 静态分析 + agent 交叉验证

**模式**:先用确定性工具扫一遍(几何/文本/统计),拿到硬清单;再派 agent 读源码验证(确认真假、补漏)。

**实例**:
- SVG 几何:`scripts/svg_audit.py` 算出文字越界/框重叠/线条穿字 → agent 读源码确认是否真问题(transform 误报排除)。
- AI 味黑名单:用 `去ai味道自查方法.md` §6 的扫描脚本跑出嫌疑词频次 → agent 按密度和上下文判定。
- 像素验证:Chrome 无头渲染 PNG → PIL 程序化检查像素(本环境无法"看图",这是唯一可靠的视觉验证法)。

**为什么有效**:工具快且准(不漏),agent 有判断力(不误报)。两者交叉才可靠。

### 3. 逆向发现(核心方法论)

**问题**:正向检查(拿黑名单扫)只能抓已知的。怎么发现**新型**问题?

**方法**:让 agent 读完全文,**专门找现有清单覆盖不到的"新型怪句"**,按"四测试"判定,归类命名。

**四测试**(判定一个句子是否"不自然"):
1. **朗读测试**:读出声来,像自然中文,还是像翻译/AI?
2. **白板测试**:一个资深工程师在白板前给同事讲架构,会**原样说**这句话吗?
3. **一遍读懂测试**:第一遍读能抓住意思,还是要回头重读?
4. **不在清单测试**:不含已知黑名单词,但就是别扭——底层是什么句式问题?

**关键**:不报"第 X 行这句话怪",而是找 3+ 个同类,**命名成新的句式病理类别**(像语言学家命名),给定义+实例+改法。

**产出**:新句式问题分类表 → 验证(3+ 实例?真新?)→ 补进 `去ai味道自查方法.md`。

### 4. 飞轮:发现 → 沉淀 → 检查 → 再发现

```
    发现(逆向)
        ↓
    沉淀(补进清单)
        ↓
    检查(正向,用新清单扫全书)
        ↓
    再发现(逆向,找更深层的)
        ↓
    ... (每轮清单更丰富,发现更深层)
```

**已沉淀的检查模式**(在 `去ai味道自查方法.md` 和本文件中):
- 已知 AI 味黑名单(框架词/互联网黑话/学术腔/程度副词/物理动作动词/翻译腔对仗/"X很Y:"起手式)。
- 造词+拔高+全称三连("三难/根本张力"类)。
- 过度抽象压缩+混合隐喻("范式对应模式/根在判断题上"类)。
- 机械三连排比收束句(段末强行三连对称)。
- 判断悬空句("这是X的根因/兑现"缺因果扣合)。
- 被动塑造腔("被X塑造"= shaped by 直译)。
- 连环设问堆叠(3+ 为什么后悬空不答)。
- 商品化隐喻泛滥("买的是/防的是"密集排比)。
- 冒号链式收口(一句话 2-3 个冒号堆结论)。

**每跑一轮发现新的,就补进来。**

## 约束

- **本环境无法看图**:Read PNG 只返回 CDN URL(403),analyze_image 也拿不到。视觉验证靠**像素渲染 + PIL 检查**或**源码坐标推理**。
- **改字词不改意思**:文风优化只改表达方式,不增删信息。
- **三家已统一**:全书"三家"已清零,标题/图注用"三款软件"、正文用"它们"。
- **破折号已收敛**:全书"——"已压到个位数(仅章标题/金句/表格占位)。
- **不碰 outline.md**:它是草稿,build 不读、读者看不到。
- **dist/ 不在 git 里**:`dist/` 目录被全局 gitignore 排除,改动不进 git,但 `架构之道.html`(单页)是 tracked 的。
