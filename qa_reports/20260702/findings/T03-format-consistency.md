## 详细信息

### 格式断裂（影响阅读体验的严重不一致）

#### 1. 图注格式：全书存在 5 种格式

本书没有统一的正式图注格式。各章各自为政，仅将图注文本放在 SVG 图下一行，但分隔符和是否加斜体（`* *`）各不相同。

**格式分布表**（总计 53 条以 `图 N-M` 开头的行）

| 格式 | 分隔符 | 斜体 | 出现位置 | 数量 | 说明 |
|------|--------|------|----------|------|------|
| `*图 N-M：...*` | 全角冒号 `：` | 是 | ch1(L29, L204) | 2 | 章内仅部分图 |
| `*图 N-M　...*` | 全角空格 `　` | 是 | ch1(L127), ch4(L39), ch5(全章7个) | 9 | 章内仅部分图 |
| `图 N-M　...` | 全角空格 `　` | 否 | ch2(全章), ch3(6个), ch7(5), ch9(7) | 18 | 主力格式之一 |
| `图 N-M：...` | 全角冒号 `：` | 否 | ch6(L18, L77, L96, L115) | 4 | 独一家 |
| `图 N-M ...` | 半角空格 ` ` | 否 | ch8(7个+1正文), ch10(3个) | 10 | 主力格式之二 |

**最严重的问题——ch1 自身内部不一致**：

同一章内，fig-1-1 和 fig-1-2 用全角冒号分隔，fig-1-3 用全角空格：

```
ch1 L29:   *图 1-1：后端基础设施的三种核心范式与代表软件...*
ch1 L127:  *图 1-3　三款软件定位雷达...*
ch1 L204:  *图 1-2：全书章节地图...*
```

**第二严重——斜体（`* *`）有无难统一**：ch1/ch4/ch5 的图注用斜体，其余 7 章不用。斜体在 HTML 渲染中等于 `<em>` 或 `<i>`，视觉效果与正文无异，但标记方式不同。

**建议统一为**：`*图 N-M 标题*`（斜体 + 半角空格），或按本书图表约定中 "`图 N-M`（第 N 章第 M 张图）" 的表述风格统一为 `图 N-M 标题`（不加斜体，半角空格）。

具体来说，这是 `docs/book-bible.md` 没有明确定义图注格式规范的结果——不是单章错，而是全书缺规范。

### 格式微偏（不影响阅读但建议统一）

#### 2. 中文与 backtick 间距不足（4 处）

以下位置中文文字紧贴 backtick 包裹的代码/术语，按中文排版习惯应在中间加空格：

| 文件 | 行号 | 原文片段 |
|------|------|----------|
| ch6 | L54 | `由`用户名@主机名`两部分` |
| ch6 | L54 | `用户名@主机名`两部分组成`（同一行双向紧贴） |
| ch7 | L73 | `到 0–16383 这 16384 个槽，再` — 中文 `槽` 后紧跟 backtick |

这些都出现在 backtick 包裹的代码段与中文正文之间。完整写法如 `由`用户名@主机名`两部分` 应为 `由 `用户名@主机名` 两部分`。

#### 3. 水平分隔线仅出现在 ch5 和 ch10（12 处 `---`）

| 文件 | 位置 | 用途 |
|------|------|------|
| ch5 | L13, L37, L91, L145, L219, L257, L307 | 分隔各节（本章导读 → 5.1 → 5.2 → ...） |
| ch10 | L9, L35, L86, L129, L187 | 分隔各节（本章导读 → 10.1 → 10.2 → ...） |

其余 8 章（ch1–ch4, ch6–ch9）完全不使用 `---` 分隔。这不是必须统一的，但作为视觉分节标记，要么全用要么全不用。

#### 4. 正文中的图引句与图注格式无关但有些冗余

ch3 和 ch6 的正文中会在图注之前再写一遍 "图 N-M + 描述句"：

```
图 3-3 画出了完整的关闭流程。       ← 正文引句
![图 3-3 Redis 关闭流程](...)
图 3-3　Redis 关闭流程：先处理...    ← 图注（正式版）
```

ch4、ch8、ch10 则没有这种重复——正文引句不再包含 "图 N-M" 编号。两种做法皆可，全书统一即可。

### 格式亮点（统一的好的地方）

1. **表格列序全书一致**：所有三家对比表均固定为 `Redis | MySQL | Kafka` 顺序。ch6 加了版本号 `Redis（7.x）`，ch10 的归纳表用 `Redis 的关键决策` 等解释性表头——都保持了列序不变。

2. **代码块语言标注正确**：全书仅 1 个 fenced code block（ch6 L40-L42，标注为 `redis`）。所有 opening fence 均带语言标注，无未标注块。

3. **标题层级无越级**：全书顶层为 `#`（章标题），节标题用 `##`，子节用 `###`。无 `####`。三级深度一致。

4. **无序列表前缀统一**：全文仅用 `- `（减号 + 空格），未混用 `*`（星号）。

5. **引用块格式统一**：全部使用 `> ` 前缀，无其他变体。引文体例（如启示句、关键数字）一致。

6. **SUMMARY.md**：结构清晰，层级一致，各级缩进规范。

### 最影响专业感的 3 个格式问题

1. **图注格式全书不统一（P0）**：引言章承诺了 "图编号统一为 `图 N-M`"，但实际图注分布在 5 种格式下，连引言自己都没统一。读者下意识会注意到 "为什么有的图有星号有的没有"、"为什么分隔符不一样"。
2. **正文引句模式不一致（P1）**：ch3/ch6 的正文引句重复带有 "图 N-M" 编号；ch8/ch10 没有。这给人一种 "不同作者写了不同章" 的感觉。
3. **水平分隔线仅 ch5 和 ch10 使用（P2）**：起得突兀，如果不全用则这两章的分隔风格与全书格格不入。

## 审查说明

- 图注分类方式：SVG 图像（`![](...)`）之后紧跟的 `图 N-M...` 行记为"正式图注"；在正文段落中以 `图 N-M 做了什么` 开头述说的记为"正文引句"。前者需要格式统一，后者可以保留自然行文。
- 本书 `SUMMARY.md` 格式良好，仅 `目录` 页存在一项的路径与 `book.json` 入册章节目录对齐即可（非本审查范围）。
- 本审查不涉及语义、内容准确性或图表设计的正确性，仅关注 markdown 格式的一致性。
