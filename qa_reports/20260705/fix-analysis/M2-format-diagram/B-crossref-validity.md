## M2.6 —— ch08 图前导语缺失（P0）

### 核验结果：属实，8/8 张图缺导语

ch01 第 1.4.3 节明确约定：

> "每张图、每张表必有正文导语引出、正文段落解读，不允许'裸奔'。"

ch08 的 8 张图无一满足"正文导语引出"这一要求。每张图都是"段落末尾 -> 直接图片 markdown -> 图注文本"的格式，没有任何一句类似"如图 8-N 所示"或"下图展示了..."的导语句。

### 定位

| 图片 | 文件路径 | 目前格式 |
|------|---------|---------|
| fig-8-1 | `diagrams/fig-8-1.svg` (line 37) | 段落结束 -> 图片 -> `图 8-1 ...`（无导语） |
| fig-8-2 | `diagrams/fig-8-2.svg` (line 69) | 同上 |
| fig-8-3 | `diagrams/fig-8-3.svg` (line 98) | 同上 |
| fig-8-4 | `diagrams/fig-8-4.svg` (line 129) | 同上 |
| fig-8-5 | `diagrams/fig-8-5.svg` (line 144) | 同上 |
| fig-8-6 | `diagrams/fig-8-6.svg` (line 163) | 同上 |
| fig-8-7 | `diagrams/fig-8-7.svg` (line 222) | 同上 |
| fig-8-8 | `diagrams/fig-8-8.svg` (line 316) | 同上 |

### 对比：其他章节的做法

ch04 的 fig-4-2 至 fig-4-6 均有导语：

```
下图把内存淘汰的真实实现画出来，看清近似 LRU 的采样淘汰与 LFU 计数器衰减的内存布局。

![图 4-2 ...](diagrams/fig-4-2.svg)
```

ch01 的 fig-1-1、fig-1-2 也有导语：

```
下图把这种对应关系固定下来。

![图 1-1 ...](diagrams/fig-1-1.svg)
```

### 修复方案

对 ch08 每张图，在图片 markdown 前加一句导语。

建议导语句：

| 图片 | 建议导语句 |
|------|-----------|
| fig-8-1 | "下图把三款软件的数据驻留权重画在同一坐标系下。" |
| fig-8-2 | "下图展示 RDB 从魔数到 footer 的整体文件布局。" |
| fig-8-3 | "下图展示 AOF 重写 + 混合持久化的双缓冲协作时序。" |
| fig-8-4 | "下图画出 InnoDB 16KB 数据页内部的七段布局。" |
| fig-8-5 | "下图展示脏页刷盘流程与崩溃恢复路径。" |
| fig-8-6 | "下图展示一个分区如何展开为多个日志段及三件套文件结构。" |

另：ch08 的图注行缺少 `*` 斜体标记，应一并补充（与 M2.2 同类问题）。

## M2.5 —— ch01 图编号不连续（P2）

### 核验结果：属实

SVG 文件：`fig-1-1.svg`、`fig-1-2.svg`、`fig-1-3.svg` 均存在，文件编号连续无缺。

问题在**文中出现顺序**：

| 出现序号 | 图片编号 | 所在小节 | 行号 |
|---------|---------|---------|------|
| 1 | 图 1-1 | 第 1.1.1 节 三种核心范式 | 27 |
| 2 | **图 1-3** | 第 1.2.3 节 Kafka 速写结尾 | 126 |
| 3 | **图 1-2** | 第 1.4.2 节 章节地图 | 200 |

图 1-3（雷达图）出现在图 1-2（章节地图）之前。读者在第 1.2.3 节看到"图 1-3"，然后跳到第 1.4.2 节才看到"图 1-2"，造成编号跳跃困惑。

### 推荐方案：交换图编号，不动内容位置

| 当前编号 | 新编号 | 涉及文件 | 涉及引用位置 |
|---------|-------|---------|------------|
| 图 1-2 章节地图 -> | **图 1-3** | `fig-1-2.svg` -> `fig-1-3.svg` | ch01 line 200-202 |
| 图 1-3 雷达图 -> | **图 1-2** | `fig-1-3.svg` -> `fig-1-2.svg` | ch01 line 126-127 |

具体操作：
1. rename `fig-1-2.svg` 为 `fig-1-3.svg`，`fig-1-3.svg` 为 `fig-1-2.svg`
2. ch01 line 126：`![图 1-3 ...](diagrams/fig-1-3.svg)` -> `![图 1-2 ...](diagrams/fig-1-2.svg)`
3. ch01 line 127：`*图 1-3 ...*` -> `*图 1-2 ...*`
4. ch01 line 200：`![图 1-2 ...](diagrams/fig-1-2.svg)` -> `![图 1-3 ...](diagrams/fig-1-3.svg)`
5. ch01 line 202：`*图 1-2：...*` -> `*图 1-3：...*`

### 不推荐的选项

- 调整内容顺序：把雷达图移到阅读路径之后（破坏内容逻辑），或把章节地图提前到 Kafka 速写之前（破坏 §1.2 的结构完整性），都不合理。
