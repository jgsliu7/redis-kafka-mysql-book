## 必须修复项 (P0)

### 1. 商标声明完全缺失

**严重度**: P0（合规底线）

**发现**: 全书所有章节文件（00-preface.md 至 11-references.md，含 10-summary/chapter.md）中**无一处**包含注册商标符号（®）或商标归属声明。以下注册商标在正文中频繁出现但未做任何权利声明：

- **Redis** — Redis Ltd. 的注册商标
- **MySQL** — Oracle Corporation 的注册商标
- **Kafka / Apache Kafka** — Apache Software Foundation 的商标（注：Apache 项目通常用 ™ 而非 ®）
- **InnoDB** — Oracle Corporation 的商标
- **ZooKeeper / Apache ZooKeeper** — Apache Software Foundation 的商标

**修法**: 在 `chapters/00-preface.md` 末尾、"是为序。"之前，插入以下商标声明段落。

可直接插入的文本：

```markdown
## 商标声明

Redis® 是 Redis Ltd. 的注册商标。MySQL® 与 InnoDB® 是 Oracle Corporation 的注册商标。
Apache Kafka®、Apache ZooKeeper® 是 Apache Software Foundation 在美国和/或其他国家的注册商标或商标。
本书中对上述商标的使用仅用于指代对应的软件产品，不表示与上述权利人之间存在任何关联、背书或赞助关系。
书中提及的其他产品和公司名称可能为其各自所有者的商标。
```

**关于正文符号标记**：出版社惯例有两种做法：
- 宽松做法（推荐）：序言集中声明即可，正文沿用简称。合规且不影响阅读。
- 严格做法：正文首次出现时标注一次符号。例如 `Redis®`、`MySQL®`、`Kafka®`。

建议先采用宽松做法（仅序言声明），待确定出版社后再按出版社要求执行。
