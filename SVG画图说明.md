# SVG 插图说明（规范 + 检查清单 + 经验教训）

> 本文档沉淀自一次全书 52 张 SVG 插图的质量大修。那次大修用「自研几何分析器 + 每图 3 个 agent 审查」交叉验证，把真实 HIGH 级问题从 33 个降到 0。本文把过程中踩过的坑、有效的检查点、标准的修复套路固化下来，供以后画图 / 改图时复用。

## 一、本书 SVG 约定（保持风格统一）

| 项 | 约定 |
|---|---|
| 文件命名 | `fig-{章号}-{图号}.svg`，如 `fig-7-5.svg` |
| 图内标题 | SVG 源文件仍写「`图 {章号}-{图号}　{标题}`」（编号与文件名**必须一致**，全角空格分隔）供单独打开查看；**构建时由 `build_html.load_svg` 自动剥离该编号行**，成品里编号标题只在下方图注（figcaption）出现一次，避免"上图下图都标 图 X-Y"的重复。解释性副标题（如"建立本章坐标系…"）保留不剥。 |
| 根元素 | `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 W H" font-family="system-ui, -apple-system, 'PingFang SC', 'Microsoft YaHei', sans-serif">` |
| 字体 | 统一用上面的 `font-family`；代码/报文用 `font-family="monospace"` |
| 画布留白 | 内容四周留 **≥ 8px**，别顶到 viewBox 边 |
| 三家配色 | Redis = 靛 `#4F46E5`/`#EEF2FF`；MySQL = 琥珀 `#D97706`/`#FEF3C7`；Kafka = 绿 `#059669`/`#ECFDF5`；中性灰 `#6B7280`/`#F3F4F6`；强调红 `#EF4444` |
| 箭头 | `<defs>` 里定义 `<marker>`，连线用 `marker-end="url(#id)"` 引用；**定义了就要用，别留死定义**。⚠️ `orient="auto"` 的 marker，其 `<path>` 三角形**必须朝 +x 画**（如 `M0,0 L10,5 L0,10 z` 配 `refX="9" refY="5"`）；若画成朝 +y（`M0,0 L10,0 L5,9`），auto 会再转 90°，箭头横着飞（fig-5-7 踩过）。 |
| 标点 | **禁用破折号「——」**（强 AI 味），用逗号 / 冒号 / 拆句代替 |

## 二、文字宽度估算（最关键——决定文字会不会溢出框 / 画布）

SVG 文字不会自动换行、不会自动缩，**画之前必须手算它占多宽**。估算公式（按字号 `fs`）：

| 字符类型 | 单字宽 |
|---|---|
| 中文 / 全角符号（，。（）·等） | ≈ `fs` |
| ASCII（字母数字 / 半角符号） | ≈ `0.55 × fs` |
| 空格 | ≈ `0.3 × fs` |

**一段文字总宽** ≈ 逐字累加。例：`"订单 / 账户落库（事务）"`（fs=11.5）≈ 8 个全角×11.5 + 3 个半角×0.55×11.5 ≈ 134px。

**由 `text-anchor` 算文字左右边**（x 是 text 的 x 属性）：
- `start`（默认）：左 = x，右 = x + 宽
- `middle`：左 = x − 宽/2，右 = x + 宽/2
- `end`：左 = x − 宽，右 = x

**判断是否装得下**：
- 在画布内：`左 ≥ 0` 且 `右 ≤ viewBox 宽`（注意 viewBox 的 min-x 可以是负数，如 `viewBox="-16 0 ..."`，此时左 ≥ −16 即可）
- 在框内：`右 ≤ 框x + 框width − 内边距` 且 `左 ≥ 框x + 内边距`，建议内边距 **≥ 4px**

> ⚠️ **多行文本**：用 `<tspan x="..." dy="...">` 分行；分析宽度时**按最长那行算**，不要把几行拼一起算。
> ⚠️ **字面转义**：`<text>*3\r\n$3...</text>` 里的 `\r\n` 是**字面 4 个字符**不会被当成换行，会撑爆框。要换行就用 tspan。

## 三、12 类常见陷阱（全是这次真实踩到的，附案例）

### 1. 文字超出所在框
文字宽度 > 框宽。
> 案例：fig-1-1「订单 / 账户落库（事务）」128px 塞进 120px 框；fig-4-4 `innodb_flush_log_at_trx_commit`（27 个 ASCII）塞进 140px 框。
> 修法：精简文案 / 加宽框 / 缩字号 / 换行。

### 2. 文字超出画布（viewBox）
坐标算错，文字画到画布外。
> 案例：fig-5-1 左侧「交互层/逻辑层/存储层」`x=28` + `anchor=end`，左缘 = 28−36 = **−8**，溢出画布左边。
> 修法：把 viewBox 左扩（`viewBox="-16 0 776 620"`），或右移标签。

### 3. 框与框部分重叠
两个**同级**框互相压边（注意：大框套小框的「嵌套」是合法的，不算 bug）。
> 案例：fig-9-6 三个 150 宽成员框中心间距只有 105，相邻重叠 45px；fig-9-1 四个子问题框同理。
> 修法：缩框宽 或 拉开间距，使「框宽 ≤ 中心间距 − 间隙」。

### 4. 文字与文字互压
两段文字在同一 y 带、x 区间重叠。
> 案例：fig-10-2 叶子框底部「→ MySQL 双1…」与下一层分支标签「偏 CP」挤在同一行；fig-6-3「mysql.tables_priv」与右侧「作用域」漏斗标注。
> 修法：把其中一段挪到层间空白 / 缩短 / 改 y 错开。

### 5. 线条 / 箭头穿过文字或框
连线没绕开内容。
> 案例：fig-5-6 两条返回响应虚线斜穿过存储层的「offset 定位」「sendfile」实心子框。
> 修法：改成沿外侧绕行的折线路径（`<path d="M ... L ... L ...">`），避开子框区。

### 6. 箭头终点错位
箭头指进框内部、悬空、或没指到目标。
> 案例：fig-3-6 红色虚线箭头终点 `x=760` 穿进红框（框左边 730）；fig-3-3 一根主线箭头指进两个分支框之间的空白（该删）；fig-8-3 箭头⑥起点离源框 70px 悬空。
> 修法：终点贴目标框边缘外侧 2–5px；指向空白的冗余箭头直接删；悬空箭头补上到源框的连线。

### 7. 定义了箭头却没用 / 连线没箭头
`<defs>` 有 `<marker>` 但连线没引用，或反过来。
> 案例：fig-2-2 生产者/消费者两根连线**没有箭头**（marker 定义了没用）；fig-2-3 定义了 arrow marker 却无任何 line 引用（死定义）。
> 修法：给该有方向的连线加 `marker-end`；没人用的 marker 定义删掉。

### 8. Ghost / 死元素
零高框、`fill="none"` 无描边框、`stroke-width="0"` 的线——看不见但污染坐标分析、可能误导。
> 案例：fig-6-2 认证区有 3 个 ghost 元素（`height="0"`、`fill="none"` 无 stroke、`stroke-width="0"`）。
> 修法：直接删掉。

### 9. 坐标语义错位
元素画在了与含义不符的位置。
> 案例：fig-4-3 midpoint 红线画在 `x=540`，但 New/Old 子列真实边界在 `x=700`（New 340–700，Old 700–916）——红线标错了分界；fig-5-4 分发箭头落在引擎框左边缘而非中心。
> 修法：按真实几何重算坐标，红线/箭头对准真实边界 / 框中心。

### 10. 图内标题编号与文件名不一致
> 案例：fig-7-5 内部标题写「图 7-2」，文件名却是 fig-7-5（重命名时漏改）。
> 修法：标题编号必须 = 文件名编号。可用脚本批量校验（见第五节）。

### 11. 内容性 bug（图与书不一致）
> 案例：**fig-1-2 章节地图漏画第 2 章**（加 Ch2 时地图没更新），且图内「七章 / 八章」自相矛盾。
> 修法：改结构图后，回头核对该图是否还覆盖全部章节 / 编号是否一致。这类 bug 几何分析抓不到，靠 agent 读源码 + 人工核对。

### 12. AI 味标点
中文标签 / 注释里的破折号「——」。
> 修法：全删，换成中文逗号或重写。可用 `grep -rn "——" chapters/**/diagrams/*.svg` 扫描。

### 13. 弧线/曲线 + 箭头头：线从箭头底的「侧沿」偏心进入（极易看错成"没接上"）
带 `marker-end` 的**曲线**箭头：箭头头按「端点处的切线方向」摆放，它的底（尾）落在端点后方约 6–7px（垂直于切线）。如果曲线在「底的位置」**仍在拐弯**（还没顺着切线方向拉直），线就会从箭头底的**侧边**穿过去，而不是从底中心进入——视觉上就是"线没接到箭头头中间/接到了边上"。
> 案例：fig-8-4 头尾 LSN 弧线、fig-7-2 迁移弧线，都是曲线在端点附近仍在拐弯，线偏到箭头底的下沿/右沿。
> 注意：这**不是**虚线缝隙问题——marker 三角形向后覆盖 6–7px，虚线 gap(≤3px)其实被盖住了，看不到。别误诊。
> 诊断（像素法，本书环境无视觉时最可靠）：高分辨率渲染，用 PIL 检查箭头底所在列/行的红色像素，看线是否落在底的中心还是偏到一侧。
> 修法：让路径在**最后 10–15px 拉直**（顺着箭头方向），使线到达底时居中。
> - 水平箭头：曲线提前到目标 y，加 `L 终点x 终点y`（水平段）。如 `C ...,115 113 L 135 113`。
> - 垂直箭头：曲线提前到目标 x，加竖直 `L`。如 `Q 470 290 470 275 L 470 265`。

## 四、每张图交稿前自检清单

逐条过一遍（几何项可用 `python3 scripts/svg_audit.py` 自动查）：

- [ ] **文字不溢出画布**：所有文字左 ≥ viewBox.min-x，右 ≤ viewBox 宽
- [ ] **文字不溢出框**：每段框内文字左右内边距 ≥ 4px（含中文按全角算宽）
- [ ] **框不部分重叠**：同级框之间无压边（嵌套除外）
- [ ] **文字不互压**：无两段文字在同 y 带、x 区间重叠
- [ ] **线条不穿字 / 穿框**：连线绕开文字和实心子框
- [ ] **箭头齐全且方向对**：该有箭头的连线都有 `marker-end`；箭头指向正确的目标元素边缘
- [ ] **无 ghost 元素**：无零高框 / 不可见框 / 零宽线
- [ ] **坐标语义正确**：分界线、指针、标注画在真实位置
- [ ] **图内标题编号 = 文件名编号**
- [ ] **留白均衡**：内容四周 ≥ 8px，左右 / 上下留白不过度失衡
- [ ] **无破折号「——」**
- [ ] **XML 合法**：`python3 -c "import xml.etree.ElementTree as ET; ET.parse('该图.svg')"`
- [ ] **能渲染**：用 Chrome headless 渲一张 PNG 确认不报错（见第六节）

## 五、审查方法（这次验证有效的流程）

### 1. 静态几何分析（确定性，快）
`python3 scripts/svg_audit.py` 自动查上面清单的几何项，输出每张图的 high/medium 问题。
- **关键：分析器必须支持 `<g transform="translate">` 累积**，否则把组内局部坐标当全局坐标，会疯狂误报（这次 fig-9-4 一张图被误报 27 处，全是 transform 没算）。
- **旋转文本**（`transform="rotate(...)"`）要特殊处理：旋转 90° 后，文字横向只占字号宽、纵向占文本长，不能套用水平宽度公式。
- 负 viewBox 原点（`viewBox="-16 ..."`）要记得减去 min-x 再判溢出（曾因漏算把 fig-5-1 合法的"交互层/逻辑层/存储层"标签误报成左溢出）。
- **v5 新增（专治"线/箭头衔接"这类低级错误，旧版几何审计完全漏检）**：`[A]` marker 朝向与 `orient=auto` 冲突（三角形朝 +y 配 auto → 箭头横飞）、`[B]` 箭头端点大幅悬空（离任何框/文字 >18px）、`[C]` 线段穿过文字、`[D]` 带箭头连线横穿框内部（框既非源也非目标）、`[E]` 图内残留编号标题（应由构建剥离）。这五类正是肉眼易漏、几何脚本能稳定抓的。

### 2. 多 agent 源码审查（抓几何抓不到的语义问题）
每张图派 **3 个 agent**，分别从 **布局 / 文字 / 线条** 三个视角读 SVG 源码判定。agent 看不到渲染图没关系——源码坐标足够做几何与逻辑推理。agent 的增量价值在于抓「几何分析抓不到」的问题：
- 箭头逻辑（方向、是否连对、是否悬空）
- 内容一致性（如章节地图是否覆盖全部章节）
- ghost 元素、死定义
- 视觉拥挤、留白失衡的主观判断

> 本环境下 SVG 质检**双保险**（2026-06 验证通）：
> 1. **真看图**：`python3 scripts/svg_render.py <图>` 用 playwright 把 SVG 渲成 PNG，再用 Read 工具**实际看到画面**验证（裁剪 `--crop x,y,w,h` 可放大局部）。旧说"看不了图"已过时——这条路通了。
> 2. **几何 + 语义**：`svg_audit.py`（含 v5 的 [A]–[E] 连接类检查）抓几何；多 agent 读源码抓语义。两者交叉印证后才定论。

### 3. 交叉验证
静态分析的发现交给 agent 判定 REAL / FALSE_POSITIVE；agent 的新发现再用静态分析复核。两者都点头才改。

### 4. 改完必验
- 重跑 `svg_audit.py`：确认该图 high 清零、无新增回归
- 重渲 PNG：确认 XML 没改坏、能正常渲染
- 重建 HTML（`scripts/build_html.py` 单页 + `scripts/build_pages.py` 分章）

## 六、常用命令速查

```bash
# 几何审计全部图
python3 scripts/svg_audit.py

# XML 合法性（全部）
for s in $(find chapters -name "*.svg"); do \
  python3 -c "import xml.etree.ElementTree as ET; ET.parse('$s')" || echo "BAD: $s"; done

# 扫描破折号
grep -rn "——" chapters/**/diagrams/*.svg

# 校验图内标题编号 = 文件名编号
python3 - <<'PY'
import glob, re, os
for s in sorted(glob.glob('chapters/**/diagrams/*.svg', recursive=True)):
    fn = os.path.basename(s)[4:-4]          # fig-7-5 -> 7-5
    t = open(s, encoding='utf-8').read()
    m = re.search(r'<text[^>]*font-weight=["\']?700["\']?[^>]*>图\s*\d+　?[^<]*', t)
    internal = re.search(r'图\s*(\d+)[-－](\d+)', m.group(0)) if m else None
    internal = f"{internal.group(1)}-{internal.group(2)}" if internal else '?'
    if internal != fn: print(f"MISMATCH {s}: 文件={fn} 内部={internal}")
PY

# Chrome headless 渲一张图确认能渲染
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless --disable-gpu --no-sandbox --window-size=1200,900 \
  --screenshot=/tmp/check.png "file://$PWD/chapters/xx/diagrams/fig-x-y.svg"

# 重建 HTML
python3 scripts/build_html.py      # 单页 架构之道.html
python3 scripts/build_pages.py     # 分章 dist/
```
